LibreOffice 25.2 讀我檔案
======================================================================


若要取得讀我檔案的最新更新，請見 https://git.libreoffice.org/core/tree/master/README.md

本檔案包含有關 LibreOffice 軟體的重要資訊。建議您在開始安裝前，請仔細閱讀本資訊。

LibreOffice 社群負責本產品的開發，我們誠摯邀請您參與我們的社群。若您是新手使用者，您可以參訪 LibreOffice 網站，該網站有許多和 LibreOffice 專案及其社群的相關資訊。請前往 https://www.libreoffice.org/。

LibreOffice 真的是任何人都能自由使用？
----------------------------------------------------------------------

LibreOffice 可以給任何人自由使用。只要您喜歡，都可以把這份 LibreOffice 安裝在任何電腦上；並且您可以採任何想要的用途來使用它 (包括商業、政府、公共行政和教育用途使用)。若要瞭解進一步的細節，請查看此 LibreOffice 下載包內的授權條款書面聲明。

為什麼 LibreOffice 可以讓任何人免費使用？
----------------------------------------------------------------------

您可以免費使用這份 LibreOffice，因為個人貢獻者與企業贊助者已設計、開發、測試、翻譯、編寫文件、提供支援、進行行銷，以及透過其他協助方式讓 LibreOffice 有今日的樣貌——開放源碼的世界領導生產力軟體，供家庭與辦公使用。

若您想感謝他們的努力，並想要確保 LibreOffice 在未來也能繼續存在的話，請考慮對本專案盡一份心力——請見 https://www.documentfoundation.org/contribution/ 以瞭解細節。任何人都能做出屬於自己的貢獻。

安裝注意事項
----------------------------------------------------------------------

LibreOffice 需要近期的 Java 執行時期環境 (JRE) 版本才能執行全部的功能。JRE 不屬於 LibreOffice 安裝包的一部分，它應該個別安裝。

系統需求
----------------------------------------------------------------------

* Microsoft Windows 7 SP1、8、8.1 Update (S14) 或 10

請注意，安裝程序需要管理員權限。

執行安裝程式時使用以下命令列開關，可強制或不讓 LibreOffice 註冊為 Microsoft Office 格式的預設應用程式：

* REGISTER_ALL_MSO_TYPES=1 會強迫 LibreOffice 註冊為 Microsoft Office 格式的預設應用程式。
* REGISTER_NO_MSO_TYPES=1 會阻止 LibreOffice 註冊為 Microsoft Office 格式的預設應用程式。

請確定您系統上的暫存目錄內有足夠的記憶體，並且確保它提供讀、寫、執行的存取權。在開始安裝程序之前，請先關閉所有其他程式。

在以 Debian/Ubuntu 為基礎的 Linux 系統上安裝 LibreOffice
----------------------------------------------------------------------

至於如何安裝語言包的指引 (在安裝完 LibreOffice 的 US English 版之後)，請閱讀下方「安裝語言包」的小節。

解壓縮下載的封存檔時，您會看到內容已解壓縮成一個子目錄。開啟檔案管理員視窗，將目錄變更為以「LibreOffice_」開頭的目錄，接著是版本號碼和一些平台資訊。

此目錄包含名為「DEBS」的子目錄。請切換到「DEBS」目錄。

對目錄點按滑鼠右鍵，並選擇「以終端機開啟」，接著將開啟終端機視窗。在終端機視窗的命令列內，輸入下列命令 (在命令執行之前，會提示您輸入 root 使用者密碼)：

下列命令將會安裝 LibreOffice 與桌面整合軟體包（您可以直接複製並貼到終端機畫面，而不用嘗試輸入）：

sudo dpkg -i *.deb

安裝程序現在已完成，您應該可以在您桌面的「應用程式」/「辦公」選單內找到所有 LibreOffice 應用程式的圖示。

在 Fedora、openSUSE、Mandriva 與其他採用 RPM 軟體包的 Linux 系統上安裝 LibreOffice
----------------------------------------------------------------------

至於如何安裝語言包的指引 (在安裝完 LibreOffice 的 US English 版之後)，請閱讀下方「安裝語言包」的小節。

解壓縮下載的封存檔時，您會看到內容已解壓縮成一個子目錄。開啟檔案管理員視窗，將目錄變更為以「LibreOffice_」開頭的目錄，接著是版本號碼和一些平台資訊。

此目錄包含名為「RRMS」的子目錄。請切換至「RPMS」目錄。

對目錄點按滑鼠右鍵，並選擇「以終端機開啟」，接著將開啟終端機視窗。在終端機視窗的命令列內，輸入下列命令 (在命令執行之前，會提示您輸入 root 使用者密碼)：

若是以 Fedora 為基礎的系統：sudo dnf install *.rpm

若是以 Mandriva 為基礎的系統：sudo urpmi *.rpm

若是其他以 RPM 為基礎的系統 (如 openSUSE 等)：rpm -Uvh *.rpm

安裝程序現在已完成，您應該可以在您桌面的「應用程式」/「辦公」選單內找到所有 LibreOffice 應用程式的圖示。

另外，您也可以改用「install」命令稿。它放在此封存檔內的最頂層目錄中，能以使用者身份進行安裝。這份命令稿會讓 LibreOffice 擁有針對此安裝的自身個人設定檔，獨立於您的一般 LibreOffice 個人設定檔。請注意，它不會安裝與系統整合的部份，例如選單項目與桌面 MIME 類型註冊等。

給以上安裝指示有關桌面整合資訊中沒有提及的 Linux 散布版之注意事項
----------------------------------------------------------------------

在本安裝指引內未特別提及的其他 Linux 散布版上安裝 LibreOffice 應該也很容易，但可能遇到的主要的差異大概是桌面整合這項。

RPM（或 DEB）目錄也包含名為 libreoffice25.2-freedesktop-menus-25.2.0.1-1.noarch.rpm（或 libreoffice25.2- debian-menus_$ {ProductVERSION}.0.1-1_all.deb 類似的名稱）。這是一個適用於所有支援 Freedesktop.org 規範/建議的Linux 散佈版的軟體包(https://zh.wikipedia.org/wiki/Freedesktop.org），並提供在上述說明中未涵蓋的其他 Linux 散佈版上安裝。

安裝語言包
----------------------------------------------------------------------

下載您要的語言與平台的語言包。這些語言包和主安裝封存檔一樣放在同個下載位置上。在 Nautilus 檔案管理員內將下載的封存檔解開至某目錄 (例如您的桌面)。請確認您已經離開所有 LibreOffice 應用程式 (包括「快速啟動」，如果有開啟的話)。

將目錄切換至您下載的語言包所解開的目錄。

現在變更目錄至解壓縮過程中建立的目錄。例如，對於 32 位元 Debian/Ubuntu 系統的法語語言包，目錄名稱為「LibreOffice_」，加上一些版本資訊，再加上 Linux_x86_langpack-deb_fr。

現在請切換至包含欲安裝軟體包的目錄。以 Debian/Ubuntu 為基礎的系統上，目錄會是 DEBS；而 Fedora、openSUE、Mandriva 等系統上，目錄則會是 RPMS。

在 Nautilus 檔案管理員內，對目錄點按滑鼠右鍵，並選擇「以終端機開啟」，接著將開啟終端機視窗。在您剛開啟的終端機視窗內，執行用來安裝語言包的命令 (在命令執行之前，會提示您輸入 root 使用者密碼)：

以 Debian/Ubuntu 為基礎的系統：sudo dpkg -i *.deb

若是以 Fedora 為基礎的系統：su -c 'dnf install *.rpm'

若是以 Mandriva 為基礎的系統：sudo urpmi *mandriva*.rpm

其他採用 RPM 的系統 (如 openSUSE 等)：rpm -Uvh *.rpm

現在，您可以啟動任何一個 LibreOffice 應用程式，例如 Writer。然後到「工具」選單並選擇「選項」。在「選項」對話方塊中，點按「語言與區域設定」然後點按「一般」。下拉「使用者介面」清單並選取您剛安裝的語言。若需要，請對「區域設定」、「預設貨幣」與「文件的預設語言」執行相同的操作。

在調整完這些設定值後，請點按「確定」。對話窗將關閉，而您會看見一道訊息告訴您說您的變更將在您離開 LibreOffice 後啟用，接著請重新啟動它 (請記住也要離開「快速啟動」功能，如果有開啟的話)。

下次您啟動 LibreOffice 時，它將會以您剛安裝好的語言啟動。

----------------------------------------------------------------------
程式啟動期間發生問題
----------------------------------------------------------------------

LibreOffice 難以啟動（例如應用程式當掉）以及螢幕顯示問題通常是顯示卡驅動程式造成。若問題持續，請更新您的顯示卡驅動程式或嘗試使用由您的作業系統提供的顯示卡驅動程式。

----------------------------------------------------------------------
Windows 中的 ALPS/Synaptics 筆記型電腦觸控板
----------------------------------------------------------------------

由於 Windows 驅動程式發生問題，您無法在 ALPS/Synaptics 觸控板上滑動手指時捲動 LibreOffice 文件。

若要啟用觸控板捲動功能，請將以下行加入「C:\Program Files\Synaptics\SynTP\SynTPEnh.ini」配置檔案，然後重新啟動電腦：

[LibreOffice]

FC = "SALFRAME"

SF = 0x10000000

SF |= 0x00004000

配置檔案的位置會隨 Windows 版本而異。

----------------------------------------------------------------------
捷徑鍵
----------------------------------------------------------------------

LibreOffice 中只能使用作業系統未使用的捷徑鍵 (按鍵組合)。如果 LibreOffice 中的按鍵組合執行方式與 LibreOffice 說明中的說明不相符，請檢查作業系統是否已使用該捷徑。若要修正此衝突，您可以變更作業系統所指定的按鍵。或是變更 LibreOffice 中幾乎所有的按鍵指定。如需有關此主題的更多資訊，請參閱 LibreOffice 說明或作業系統的說明文件。

----------------------------------------------------------------------
當從 LibreOffice 以文件傳送電子郵件時發生問題
----------------------------------------------------------------------

透過「檔案」→「 傳送」→「以文件傳送電子郵件」或「檔案」→「 傳送」→「以 PDF 傳送電子郵件」傳送文件時可能會出現問題（程式崩潰或無回應）。這是因為 Windows 系統檔案「Mapi」（Messaging Application Programming Interface）導致某些檔案版本出現問題。不幸的是，問題無法縮小到某個版本號。要取得詳細資訊，請造訪 https://www.microsoft.com，並在 Microsoft 知識庫中搜尋「mapi dll」。
